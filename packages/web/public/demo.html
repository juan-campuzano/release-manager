📡 This is a Demo UI

The actual application is a REST API backend. To interact with real data:

1. Start the server:

npm run build && npm start

2. Create a release via API:

curl -X POST http://localhost:3000/api/releases \
  -H "Content-Type: application/json" \
  -d '{
    "platform": "iOS",
    "version": "1.0.0",
    "branchName": "release/1.0.0",
    "repositoryUrl": "https://github.com/your-org/repo",
    "sourceType": "github",
    "requiredSquads": ["mobile", "qa"],
    "qualityThresholds": {
      "crashRateThreshold": 1.0,
      "cpuExceptionRateThreshold": 0.5
    },
    "rolloutStages": [1, 10, 50, 100]
  }'

3. View releases:

curl http://localhost:3000/api/releases

Available Endpoints:

  • GET /api/health - Health check
  • GET /api/releases - List all releases
  • POST /api/releases - Create release
  • GET /api/releases/:id - Get specific release
  • POST /api/releases/:id/blockers - Add blocker
  • POST /api/releases/:id/signoffs - Record sign-off

📖 See USAGE.md for complete API documentation
